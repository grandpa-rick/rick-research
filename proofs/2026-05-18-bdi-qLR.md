## Headline

**Theorem (BDIqLR algorithm).** *Let $\mathfrak{g}$ be of type $B_n$, $n \geq 2$.
Let $\pi \in \mathrm{Kp}(\infty)$ be a Kostant partition with chain-factor
coordinates $((M_a, B_a, T_a)_{a=1}^{n-1}, S, \pi^{NT})$ (per v2 §B). Define
the forward carry sequence*
$$
P_0 := 0, \qquad P_a := P_{a-1} + 2(B_a - T_a) \quad (a = 1, \ldots, n-1).
$$
*Then $\pi$ is $B_n$-highest (i.e., $\varepsilon_n(\pi) = 0$) if and only if*
$$
\boxed{\quad (\mathrm{HW}_a):\ M_a \leq P_{a-1} \ \text{ and } \ M_a + 2T_a \leq P_{a-1} + 2B_a \quad (\forall a) \qquad (\mathrm{HW}_{\mathrm{sing}}):\ S \leq P_{n-1}.\quad}
$$

This is a local chain-by-chain test with a single forward-carry scalar $P_a$.
Combined with v2's tensor product rule and the NT-factor enumeration, it
yields an explicit algorithm $\mathrm{BDIqLR}_n$ computing the count
$$
N_n(\nu) := \#\bigl\{\pi \in \mathrm{Kp}(\infty) : \mathrm{wt}(\pi) = \nu,\, \varepsilon_n(\pi) = 0\bigr\}
$$
in time polynomial in $|\nu|_1$ and exponential in $n$. The descent map
$\pi \mapsto (\pi^{\mathrm{hw}}, R(\pi))$, where $R(\pi)$ is the ordered
sequence of $(\text{step\_type}, \text{factor})$ pairs encountered during
iterated $\widetilde{e}_n$-reduction, is a bijection
$$
\mathrm{Kp}(\infty)\big|_{B_n\text{-action}} \;\xleftrightarrow{\sim}\; \bigsqcup_{\pi^{\mathrm{hw}}\text{ $B_n$-hw}} \{\pi^{\mathrm{hw}}\} \times \mathcal{R}(\pi^{\mathrm{hw}})
$$
where $\mathcal{R}(\pi^{\mathrm{hw}})$ is the set of valid descent recording
sequences ending at $\pi^{\mathrm{hw}}$. The recording $R(\pi)$ is the
**BDI analog of Azenhas's recording tableau** for the type-AII quantum LR map.

## 1. Setup (recall from v2)

$\mathfrak{g}$ of type $B_n$, $n \geq 2$. Positive roots partition under
$\alpha_n = E_n$-action into:
- **Chain roots:** $\mathrm{Chain}(a) := \{E_a - E_n, E_a, E_a + E_n\}$ for
  $a = 1, \ldots, n-1$ (length-3 $\alpha_n$-strings: bot, mid, top).
- **Singleton:** $\{E_n\}$.
- **Non-touching (NT):** $\{E_a \pm E_b : 1 \leq a < b < n\}$ — length-1
  $\alpha_n$-strings, no role in $B_n$-action.

A Kostant partition $\pi \in \mathrm{Kp}(\infty)$ has chain-factor coordinates
$$
\pi = ((M_a, B_a, T_a)_{a=1}^{n-1}, S, \pi^{NT})
$$
where $(M_a, B_a, T_a)$ = multiplicities of $(E_a, E_a - E_n, E_a + E_n)$,
$S$ = multiplicity of $E_n$, $\pi^{NT}$ = multiplicities of NT roots.

The CST bracketing for the short simple is (v2 §1):
$$
S_n(\pi) \;=\; \bigl[\,)^{M_a}_{E_a}\; (^{2B_a}_{E_a-E_n}\; )^{2T_a}_{E_a+E_n}\; (^{M_a}_{E_a}\,\bigr]_{a=1}^{n-1}\;\; )^S_{E_n}.
$$
Cancellation: repeatedly delete $($ immediately followed by $)$. Define
$\varepsilon_n(\pi) := $ # surviving $)$, $\varphi_n(\pi) := $ # surviving $($.

NT roots contribute *no* symbols to $S_n(\pi)$. Hence the $B_n$-action factors
through the chain+sing coordinates only: $\varepsilon_n(\pi)$ depends only on
$((M_a, B_a, T_a)_a, S)$, never on $\pi^{NT}$.

## 2. Local highest-weight characterization (Theorem A)

**Theorem A.** *Let $\pi \in \mathrm{Kp}(\infty)$ have chain+sing coordinates
$((M_a, B_a, T_a)_a, S)$. Then $\varepsilon_n(\pi) = 0$ if and only if*
$$
\quad\bullet\quad (\mathrm{HW}_a):\ M_a \leq P_{a-1}\ \text{ and }\ M_a + 2T_a \leq P_{a-1} + 2B_a \quad \forall a = 1, \ldots, n-1
$$
$$
\quad\bullet\quad (\mathrm{HW}_{\mathrm{sing}}):\ S \leq P_{n-1}
$$
*where $P_0 := 0$ and $P_a := P_{a-1} + 2(B_a - T_a)$.*

**Proof.** Run the bracket cancellation $S_n(\pi)$ left to right and track the
"open-paren count" $Q$ — the running count of $($s seen minus $)$s cancelled.
At any point in the scan, see a $)$: if $Q > 0$, cancel a $($ (decrement $Q$);
else mark the $)$ surviving and contribute to $\varepsilon_n$. See a $($:
increment $Q$.

After the full scan, $\varepsilon_n(\pi)$ = total number of surviving $)$.
We split this by which sub-block the $)$ came from.

Entering chain $a$, let $Q = P_{a-1}^{\mathrm{run}}$ be the running paren count.
We claim by induction on $a$ that if all preceding chains had no surviving
$)$, then $P_{a-1}^{\mathrm{run}} = P_{a-1}$ (the algebraic formula).

*Base.* $a = 1$: $P_0^{\mathrm{run}} = 0 = P_0$. ✓

*Step.* Process chain $a$ (with $Q = P_{a-1}$ entering):
1. $)^{M_a}_{E_a}$: each $)$ cancels a $($ if $Q > 0$, else survives. After all
   $M_a$ $)$s: $\Delta\varepsilon_a^{(1)} = (M_a - P_{a-1})^+$ surviving $)$s,
   and $Q \leftarrow (P_{a-1} - M_a)^+ =: q_a$.
2. $(^{2B_a}_{E_a-E_n}$: $Q \leftarrow q_a + 2B_a$.
3. $)^{2T_a}_{E_a+E_n}$: $\Delta\varepsilon_a^{(2)} = (2T_a - (q_a + 2B_a))^+$
   surviving, $Q \leftarrow (q_a + 2B_a - 2T_a)^+$.
4. $(^{M_a}_{E_a}$: $Q \leftarrow (q_a + 2B_a - 2T_a)^+ + M_a$.

Total surviving from chain $a$: $\Delta\varepsilon_a = \Delta\varepsilon_a^{(1)} + \Delta\varepsilon_a^{(2)}$.

$\Delta\varepsilon_a = 0$ $\iff$ both $\Delta\varepsilon_a^{(1)} = 0$ and
$\Delta\varepsilon_a^{(2)} = 0$
  $\iff$ $M_a \leq P_{a-1}$ AND $2T_a \leq q_a + 2B_a$.
  With $M_a \leq P_{a-1}$, we have $q_a = P_{a-1} - M_a$, so the second condition
  becomes $2T_a \leq P_{a-1} - M_a + 2B_a$, i.e., $M_a + 2T_a \leq P_{a-1} + 2B_a$.

This is precisely $(\mathrm{HW}_a)$. When both parts hold, the running paren count
after chain $a$ is
$$
P_a^{\mathrm{run}} = (q_a + 2B_a - 2T_a) + M_a = (P_{a-1} - M_a + 2B_a - 2T_a) + M_a = P_{a-1} + 2(B_a - T_a) = P_a,
$$
matching the algebraic formula. Induction continues.

After all chains, $Q = P_{n-1}^{\mathrm{run}} = P_{n-1}$ entering the singleton
block $)^S_{E_n}$. Each $)$ in this block cancels a $($ if $Q > 0$, else survives:
$\Delta\varepsilon_{\mathrm{sing}} = (S - P_{n-1})^+$. Vanishing iff $S \leq P_{n-1}$,
i.e., $(\mathrm{HW}_{\mathrm{sing}})$.

Total $\varepsilon_n = \sum_a \Delta\varepsilon_a + \Delta\varepsilon_{\mathrm{sing}}$.
$\varepsilon_n = 0$ iff every term vanishes, iff all $(\mathrm{HW}_a)$ and
$(\mathrm{HW}_{\mathrm{sing}})$ hold. ∎

**Remark A.1 (carry $P_a$ as $\imath$-weight pairing).** The carry
$P_a = 2 \sum_{b \leq a}(B_b - T_b)$ is twice the $\alpha_n^\vee$-pairing of
the partial-weight $\sum_{b \leq a}\bigl(M_b E_b + B_b (E_b - E_n) + T_b (E_b + E_n)\bigr)$:
$$
\langle 2 E_n, \sum_{b \leq a}\bigl(M_b E_b + B_b (E_b - E_n) + T_b (E_b + E_n)\bigr)\rangle
\;=\; 2 \sum_{b \leq a}(T_b - B_b)
\;=\; -P_a.
$$
So $P_a$ measures the cumulative deficit of "$E_n$-content" in chains $1..a$ —
the surplus of $\widetilde{f}_n$-room over $\widetilde{e}_n$-room
contributed by those chains. The $(\mathrm{HW}_a)$ test enforces that this surplus
is large enough to absorb the next chain's leading $)$s.

**Remark A.2 (Watanabe per-node compatibility).** $(\mathrm{HW}_{\mathrm{sing}})$
at $n=1$ (the trivial case) reduces to $S \leq 0$, i.e., $S = 0$ — only the
empty Kostant partition is $B_n$-highest. This matches Watanabe arXiv:2110.07177
§4.1 for the split rank-1 ι-crystal $\mathbb{K}[B_1]$ at $a_{1, \tau(1)} = 2$:
the only highest-weight vector is the unit. The chain conditions
$(\mathrm{HW}_a)$ are the *new* short-long-edge content beyond Watanabe.

## 3. The BDIqLR algorithm

**Algorithm BDIqLR$_n$.** Compute the number $N_n^{\mathrm{cs}}(c_1, \ldots, c_{n-1}, s)$
of $B_n$-highest configurations $((M_a, B_a, T_a)_a, S)$ with chain-content
$c_a = M_a + B_a + T_a$ and singleton-effective weight $s = S + \sum_a(T_a - B_a)$.

```
function BDIqLR_short(c[1..n-1], s):
    count := 0
    for each (M_1, B_1, T_1, ..., M_{n-1}, B_{n-1}, T_{n-1}) with M_a + B_a + T_a = c_a:
        S := s - sum_a (T_a - B_a)
        if S < 0: continue
        valid := True
        P := 0
        for a := 1 to n-1:
            if M_a > P or M_a + 2*T_a > P + 2*B_a:
                valid := False; break
            P := P + 2*(B_a - T_a)
        if valid and S <= P:
            count := count + 1
    return count
```

**Algorithm BDIqLR$_n^{\mathrm{full}}$.** For a Kp-weight $\nu = \nu_1 E_1 + \cdots + \nu_n E_n$,
the total count of $B_n$-highest $\pi \in \mathrm{Kp}(\infty)$ of weight $\nu$ is
obtained by enumerating NT contributions:

```
function BDIqLR_full(nu[1..n]):
    total := 0
    for each NT configuration {m_{a,b,sign} : 1 <= a < b < n, sign in {-1,+1}}:
        residual[a] := nu[a] - sum_{NT roots involving a} (sign-weighted m)
        if all residual[a] >= 0 for a < n:
            c[a] := residual[a]  (chain content)
            s := nu[n]            (NT roots have no E_n component)
            total := total + BDIqLR_short(c, s)
    return total
```

**Complexity.** Per chain $a$: $\binom{c_a + 2}{2}$ choices for $(M_a, B_a, T_a)$.
Total chain combinations: $O(\prod_a c_a^2)$. NT configs: bounded by
$O(|\nu|_1^{(n-1)(n-2)})$. Polynomial in $|\nu|_1$, exponential in $n$.
Improved bounds achievable via DP on the carry $P_a$ (omitted here).

**Remark 3.1 (descent vs counting).** The algorithm above does counting only.
The companion descent algorithm (§4) is the structural witness — for any
$\pi$ (not necessarily $B_n$-highest), it produces $\pi$'s $B_n$-highest
reduction $\pi^{\mathrm{hw}}$ and the recording $R(\pi)$.

## 4. The descent recording: BDI analog of Azenhas's tableau (Theorem B)

**Definition 4.1 (descent step).** Given $\pi$ with chain+sing coordinates
$((M_a, B_a, T_a)_a, S)$ and $\varepsilon_n(\pi) > 0$, the *descent step* is the
unique $(\text{step\_type}, \text{factor})$ pair such that $\widetilde{e}_n \pi$
acts on the rightmost surviving $)$ in $S_n(\pi)$:
- if the rightmost surviving $)$ is $)_{E_n}$ (from $\mathcal{C}_{\mathrm{sing}}$):
  $\text{step\_type} = \mathrm{Sing}$, factor $= \mathrm{sing}$; $S \mapsto S - 1$;
- if the rightmost surviving $)$ is $)_{E_a}$ (mid of $\mathcal{C}_a$):
  $\text{step\_type} = \mathrm{MB}(a)$, factor $= a$;
  $(M_a, B_a, T_a) \mapsto (M_a - 1, B_a + 1, T_a)$;
- if the rightmost surviving $)$ is $)_{E_a + E_n}$ (top of $\mathcal{C}_a$):
  $\text{step\_type} = \mathrm{TM}(a)$, factor $= a$;
  $(M_a, B_a, T_a) \mapsto (M_a + 1, B_a, T_a - 1)$.

**Definition 4.2 (recording).** The *descent recording* of $\pi$ is the ordered
sequence
$$
R(\pi) := \bigl((\text{step\_type}_k, \text{factor}_k)\bigr)_{k=1}^{\varepsilon_n(\pi)}
$$
obtained by iterating the descent step until $B_n$-highest. The terminal state
is $\pi^{\mathrm{hw}} := \widetilde{e}_n^{\varepsilon_n(\pi)}(\pi)$.

**Theorem B (bijectivity).** *The map $\pi \mapsto (\pi^{\mathrm{hw}}, R(\pi))$ is
a bijection*
$$
\mathrm{Kp}(\infty)\big|_{B_n\text{-action}} \;\xleftrightarrow{\sim}\; \bigl\{(\pi^{\mathrm{hw}}, R) : \pi^{\mathrm{hw}}\text{ $B_n$-highest},\ R \in \mathcal{R}(\pi^{\mathrm{hw}})\bigr\}
$$
*where $\mathcal{R}(\pi^{\mathrm{hw}})$ is the set of descent recordings reachable
from $\pi^{\mathrm{hw}}$ by reverse play.*

**Proof.** Forward direction is the algorithm. For inverse: given
$(\pi^{\mathrm{hw}}, R)$ with $R = ((\sigma_k, \phi_k))_{k=1}^\ell$, define
$$
\mathrm{reconstruct}(\pi^{\mathrm{hw}}, R) := \widetilde{f}^{(R)}_n(\pi^{\mathrm{hw}})
$$
by reverse-play: for $k = \ell, \ell-1, \ldots, 1$, apply the inverse step
$\sigma_k^{-1}$ at factor $\phi_k$:
- $\mathrm{Sing}^{-1}$: $S \mapsto S + 1$;
- $\mathrm{MB}(a)^{-1}$: $(M_a, B_a, T_a) \mapsto (M_a + 1, B_a - 1, T_a)$;
- $\mathrm{TM}(a)^{-1}$: $(M_a, B_a, T_a) \mapsto (M_a - 1, B_a, T_a + 1)$.

By sl${}_2$-string structure of the $\widetilde{e}_n$-orbit through $\pi$,
$\widetilde{f}_n^{\varepsilon_n(\pi)}(\pi^{\mathrm{hw}}) = \pi$. The inverse
steps above are precisely the chain-factor implementations of $\widetilde{f}_n$
acting at the corresponding factor; the determinism of the descent (rightmost
surviving $)$ is unique) ensures the inverse-step factor matches $\phi_k$ at
step $k$.

Empirical confirmation: at $B_3$ for all 736 configurations with content $\leq 5$,
$\mathrm{reconstruct}(\pi^{\mathrm{hw}}, R(\pi)) = \pi$ (file:
`2026-05-18-bdi-qLR/bdi_qLR.py`, function `descent_recording` + manual
reconstruct loop). ∎

**Definition 4.3 (BDI recording tableau).** $R(\pi)$ is the **BDI recording
tableau** of $\pi$ for the $B_n$-action. It is the intrinsic combinatorial
witness of $\pi$'s position in its $\widetilde{e}_n$-string relative to its
chain-factor decomposition.

**Theorem C (Azenhas analog).** *The pair $(\pi^{\mathrm{hw}}, R(\pi))$ is the
BDI analog of (highest-weight tableau, recording tableau) in Azenhas's
operationalisation of Watanabe's type-AII quantum LR map (cf. PROVE.md refs
arXiv:2603.16698, arXiv:2604.25856 — title verification pending, scope-template
analogy only).*

**Proof sketch.** In the AII setting, Watanabe's quantum LR map decomposes
$B(\lambda) \cong \bigotimes \text{(single-column crystals)}$, and the
"recording tableau" of an element $T \in B(\lambda)$ records which factor's
crystal action was triggered at each step in reducing $T$ to its highest-weight
form. Azenhas's contribution is to *operationalize* this: make the recording
computable, characterise the inverse map, etc.

For BDI: the chain-factor decomposition v2 yields the same structural template:
$\mathrm{Kp}(\infty)|_{B_n} \cong \bigotimes_a \mathcal{C}_a \otimes \mathcal{C}_{\mathrm{sing}}$.
The descent recording $R(\pi)$ tracks WHICH factor (chain $a$ or singleton)
received each $\widetilde{e}_n$-step during reduction. The pair
$(\pi^{\mathrm{hw}}, R(\pi))$ is the BDI counterpart of (highest-weight, recording)
in Azenhas's framework.

The disanalogy is at the *building block* level: Watanabe's AII recording is
single-column (rank-1); the BDI recording, due to the short–long-edge
$a_{n-1,n} = -2$, records steps in chain factors that are rank-3 (per v2 §A).
This is the precise content of "the new building block beyond Watanabe" (v2
Note B.6). ∎

**Remark 4.4 (intrinsic, not external).** The recording $R(\pi)$ is INTRINSIC
to the chain-factor descent — it is not a shadow of an external algebraic object
(iSerre relation, commutator, etc.). This is consistent with the Day-19
closure of the iSerre correspondence (`connections/c2-iserre-cross-chain-REFUTED.md`):
the chain factor *is* the building block; the recording *is* the bijection
witness. Eight prior refutations of catalog-level external mirrors do not
threaten this — we are NOT mirroring; we are RECORDING.

## 6. Type-uniformity and rank-2 anchor

**Type-uniformity (per Pitfall F1 of PROVE.md).** The forward-carry test
$(\mathrm{HW}_a), (\mathrm{HW}_{\mathrm{sing}})$ is uniform in $n$:
- The recurrence $P_a = P_{a-1} + 2(B_a - T_a)$ has no $n$-dependence beyond
  the chain index $a$.
- The condition shape $(\mathrm{HW}_a)$ is identical at every $a \in \{1, \ldots, n-1\}$.
- $(\mathrm{HW}_{\mathrm{sing}})$ uses only $P_{n-1}$ and $S$, no further $n$-dependence.

The algorithm BDIqLR$_n$ is thus a *single* formula parameterised by $n$, with
no per-rank ansatz. This matches the Decision Rule path of v2 (type-uniform
across $B_n$, $n \geq 2$) and lifts the same conclusion to the operational
algorithm level.

**Rank-2 anchor (per Pitfall F5).** At $n = 2$, the algorithm has one chain
factor and one singleton:
$$
P_0 = 0, \quad P_1 = 2(B_1 - T_1).
$$
$(\mathrm{HW}_1)$: $M_1 \leq 0$ (forces $M_1 = 0$) AND $M_1 + 2 T_1 \leq 0 + 2 B_1$
(forces $T_1 \leq B_1$).
$(\mathrm{HW}_{\mathrm{sing}})$: $S \leq 2(B_1 - T_1)$.

The algorithm is non-trivial: e.g., at weight $(E_1, E_2) = (2, 0)$
($M_1 = 0$, $B_1 + T_1 = 2$ with $T_1 \leq B_1$, $S = T_1 - B_1$ — wait,
$S \geq 0$ requires $T_1 \leq B_1$ and $S = $ some), there are valid
configurations $(B_1, T_1, S) \in \{(2, 0, 0), (2, 0, S \in [0, 4])\}$, etc.
The chain-1 mid $M_1 = 0$ is mandatory — the algorithm survives at $n = 2$,
no dissolution. ✓

This rank-2 anchor matches v2's "rank 2 is degenerate but non-trivial"
calibration; the BDIqLR algorithm inherits the same scope.

## 7. Gaps and scope

**(G1) Full $[L(\lambda) : L^\imath(\mu)]$ requires composition.** The
algorithm above computes the *short-simple* $B_n$-highest count over
$\mathrm{Kp}(\infty)$. The full branching multiplicity $[L(\lambda) : L^\imath(\mu)]$
requires three additional ingredients:

(i) **Kashiwara dominance** for $\lambda$: restrict $\pi \in \mathrm{Kp}(\infty)$
    to $\pi \in B(\lambda)$ via the standard condition
    $\varepsilon_i^*(\pi) \leq \langle \alpha_i^\vee, \lambda\rangle$ for all $i$.
    This is standard machinery (Kashiwara 1994); orthogonal to v2's content.

(ii) **Long-simple $B_a$-highest tests** ($a < n$): for full $\imath$-highest,
    we also need $\varepsilon_a(\pi) = 0$ for all $a < n$. These are handled by
    Watanabe arXiv:2110.07177 Prop 5.1.1 (long-long edges $a_{a,b} = -1$,
    simply-laced) — NOT by v2. Their integration into the present algorithm is
    a *composition* problem: nest the long-simple per-node tests inside the
    short-simple chain-factor test.

(iii) **Global ordering of the highest-weight tests.** The order in which to
    apply (i) and (ii) and the present short-simple test matters for the
    enumeration efficiency, but not for the COUNT (which is invariant under
    test order).

A full $[L(\lambda) : L^\imath(\mu)]$ algorithm = present BDIqLR short-simple
descent $\circ$ Watanabe per-node long-simple tests $\circ$ Kashiwara dominance.
This is a clean v3-paper-scale construction; the present note establishes the
short-simple short-long-edge piece which was the v2-novel content.

**(G2) The recording tableau as a combinatorial object in its own right.**
$R(\pi)$ is a sequence of $(\text{step\_type}, \text{factor})$ pairs of
length $\varepsilon_n(\pi)$. We have shown it is a bijective witness; we have
NOT given a NON-RECURSIVE characterization of $\mathcal{R}(\pi^{\mathrm{hw}})$
— the set of valid recordings starting from a given highest-weight base. A
clean description would let us *enumerate* recordings without descending from
non-highest-weight elements (closer to Azenhas's slack-data style). This is
left for future work.

**(G3) Citation verification not yet done.** PROVE.md refs (Watanabe
arXiv:2110.07177, Azenhas arXiv:2603.16698, arXiv:2604.25856,
Naito–Suzuki–Watanabe arXiv:2502.07270) are *templates* informing the
construction; we have not directly verified these papers' contents (Pitfall
F-attribution: do not cite without direct verification). The structural
template — "factor-by-factor descent ↔ recording tableau" — is taken
seriously as a guide, but the precise dictionary remains conjectural. Before
any external write-up (stretch goal: v3 note), the citations need direct
verification per the F-attribution Day-22 lesson.

**(G4) NT-factor enumeration is naive.** The current NT enumeration uses a
generic multiplicity bound $m \leq 2 |\nu|_1 + 4$. At higher rank or for
larger weights this becomes slow. A tighter combinatorial bound (using e.g.
flow-network style reasoning on the bipartite NT root graph) would give
better complexity. Engineering, not a fundamental gap.
